Day 51

was changing long to int32_t in ABL related files… well externs are evil! consider: 1.cpp int myVar; 2.cpp extern long myVar; good luck looking for a bug… ABL uses lots of them!!! even if you think you’ve correclty changed all longs to ints, there is such code as: long my_stack_var; memcpy(myVar, my_stack_var, sizeof(long)); // well this you may catch by searching for "long" // but consider this memcpy(myVar, my_stack_var, sizeof(my_stack_var));// :O I hope I did not leave such cases….
